# Federal and Aerospace AI Suite Application Blueprint — Agent Context File

The Federal and Aerospace AI Suite Application Blueprint is defined and deployed using Docker Compose files. The Blueprint combines LLM inference capability served through the OpenVINO™ model server platform, speech-to-text transcription through the Whisper service, a chat UI through the Open WebUI software, and metrics information through the Grafana dashboard; and runs with the Visual Pipeline and Platform Evaluation Tool for pipeline visualization, on Intel® edge hardware.

## Deployment Modes

| Mode         | Compose file                    | When to use                                                                 |
| ------------ | ------------------------------- | --------------------------------------------------------------------------- |
| Standard     | `docker-compose.yml`            | Production. Requires Visual Pipeline and Platform Evaluation Tool network                                        |
| CDI/SR-IOV   | `docker-compose-cdi.yml`        | Virtual and physical Function GPU passthrough (set `CDI_XE_DEVICE_0` in `.env`) |
| Standalone   | `docker-compose-standalone.yml` | Development or testing — creates its own `fedaero` network, no Visual Pipeline and Platform Evaluation Tool needed         |

**Critical**: The main and Container Device Interface (CDI) compose files declare the Visual Pipeline and Platform Evaluation Tool network as `external: true`. Running `docker compose up` directly **will fail** unless Visual Pipeline and Platform Evaluation Tool is already running. Always use `make deploy`, `make deploy-cdi`, or `make up-standalone` for development.

## Component Map

| Path                                    | Purpose                                                                               |
| --------------------------------------- | ------------------------------------------------------------------------------------- |
| `docker-compose.yml`                    | Defines and runs all services in the application using the standard GPU configuration                                         |
| `docker-compose-cdi.yml`                | Container Device Interface (CDI) and Single Root I/O Virtualization (SR-IOV) variant                                                                 |
| `docker-compose-standalone.yml`         | Development or test. No Visual Pipeline and Platform Evaluation Tool required.                                                        |
| `Makefile`                              | All lifecycle targets                                                                 |
| `.env`                                  | Runtime configuration (auto-generated by `make setup`; not committed)                        |
| `apps/LLM-OpenWebUI/`                   | OpenVINO model server and Open WebUI chat interface                                         |
| `apps/speech-to-text/`                  | Whisper speech-to-text service (Flask application, Dockerfile, Prometheus metrics, and test WAV file)             |
| `apps/speech-to-text/static/index.html` | Browser-based speech-to-text demonstration UI                                                             |
| `apps/grafana/provisioning/dashboards/` | Pre-provisioned dashboards (`panther-lake-compact-v2.json`, `panther-lake-live.json`) |
| `apps/nginx/nginx.conf`                 | HTTPS reverse proxy (self-signed cert, TLS 1.3 only)                                  |
| `apps/nginx/nginx-standalone.conf`      | HTTPS reverse proxy for standalone mode (without Visual Pipeline and Platform Evaluation Tool UI block)                     |
| `collector/telegraf.conf`               | Telegraf configuration copied into Visual Pipeline and Platform Evaluation Tool metrics-manager through `make vippet-configure`         |

## Key Environment Variables

| Variable           | Set by                     | Purpose                                   |
| ------------------ | -------------------------- | ----------------------------------------- |
| `RENDER_GID`       | `make setup` (auto-detect) | GPU device group for OpenVINO model server|
| `GRAFANA_PASSWORD` | manual / defaults `admin`  | Grafana dashboard admin password          |
| `CDI_XE_DEVICE_0`  | manual                     | CDI device path (CDI variant only)        |

Run `make setup` (idempotent) to auto-populate `.env` before any `make up` variant.

## Useful Commands

```bash
make setup            # detects the render group GID and writes it into the .env file (idempotent)
make deploy           # full one-shot deployment: sets runtime environment variables for the stack, tailors Visual Pipeline and Platform Evaluation Tool installation (metrics-manager and supported models) and brings up the stack components in the correct order
make deploy-cdi       # full one-shot: CDI and SR-IOV variant
make vippet-get       # does sparse-checkout of Visual Pipeline and Platform Evaluation Tool into ../vippet-fedaero (excludes docs)
make vippet-configure # copies telegraf.conf into the Visual Pipeline and Platform Evaluation Tool and patches the tool's compose.yml file
make vippet-up        # starts Visual Pipeline and Platform Evaluation Tool (builds images and installs models)
make vippet-down      # stops Visual Pipeline and Platform Evaluation Tool
make up               # starts this stack only (Visual Pipeline and Platform Evaluation Tool must already be running)
make up-standalone    # starts without Visual Pipeline and Platform Evaluation Tool (for development or testing)
make down             # stops all services and Visual Pipeline and Platform Evaluation Tool
make build            # builds local images (whisper-stt:latest)
make restart          # restarts all services
make test             # checks service connectivity (curl health endpoints)
make urls             # prints all endpoints
make clean CONFIRM=yes  # removes containers, volumes, and data or directories — destructive
```

## Endpoints

Open WebUI chat UI, Grafana dashboard, and Whisper speech-to-text service are only accessible via the NGINX TLS reverse proxy; their container ports are not exposed to the host. Use `-k` with curl for a self-signed certificate.

| Service                                        | URL                      | Notes                               |
| ---------------------------------------------- | ------------------------ | ----------------------------------- |
| Open WebUI chat UI                             | https://localhost:8443   | LLM chat UI (via NGINX              |
| Grafana dashboard                              | https://localhost:7443   | Dashboards (via NGINX)              |
| Whisper speech-to-text service                 | https://localhost:5443   | Speech-to-text feature (via NGINX)  |
| Visual Pipeline and Platform Evaluation Tool UI| https://localhost:443    | via NGINX (standard/CDI modes only) |
| Metrics Manager                                | http://localhost:9090    | Prometheus metrics API              |
| Metrics Manager                                | http://localhost:9273    | Telegraf metrics endpoint           |
